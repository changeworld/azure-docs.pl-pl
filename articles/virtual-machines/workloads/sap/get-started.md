# <a name="use-azure-to-host-and-run-sap-workload-scenarios"></a>Używanie platformy Azure do hostowaniu i uruchamianiu scenariuszy obciążenia SAP

Korzystając z platformy Microsoft Azure, można niezawodnie uruchamiać krytyczne obciążenia i scenariusze SAP na skalowalnej, zgodnej i sprawdzonej w przedsiębiorstwie platformie. Zapewnia skalowalność, elastyczność i oszczędność kosztów platformy Azure. Dzięki rozszerzonemu partnerstwu między firmą Microsoft i SAP można uruchamiać aplikacje SAP w scenariuszach programowania i testowania i produkcji na platformie Azure i być w pełni obsługiwane. Od SAP NetWeaver do SAP S/4HANA, SAP BI w systemie Linux do Windows i SAP HANA do SQL, mamy dla Ciebie.

Oprócz obsługi scenariuszy SAP NetWeaver z różnych dbms na platformie Azure, można obsługiwać inne scenariusze obciążenia SAP, takich jak SAP BI na platformie Azure. 

Unikatowość platformy Azure dla sap HANA to oferta, która wyróżnia platformę Azure. Aby włączyć hosting więcej pamięci i środowiska SAP wymagające zasobów procesora, które obejmują SAP HANA, Platforma Azure oferuje korzystanie z dedykowanego przez klienta sprzętu bez systemu operacyjnego. To rozwiązanie służy do uruchamiania wdrożeń SAP HANA, które wymagają do 24 TB (skalowanie w poziomie 120 TB) pamięci dla S/4HANA lub innego obciążenia SAP HANA. 

Scenariusze obciążenia usługi SAP hostowania na platformie Azure można również utworzyć wymagania dotyczące integracji tożsamości i logowania jednokrotnego. Taka sytuacja może wystąpić, gdy używasz usługi Azure Active Directory (Azure AD) do łączenia różnych składników SAP i sap software-as-a-service (SaaS) lub platform-as-a-service (PaaS) oferuje. Lista takich scenariuszy integracji i logowania jednokrotnego z jednostkami usługi Azure AD i SAP jest opisana i udokumentowana w sekcji "Integracja tożsamości SAP AAD i logowanie jednokrotne".
