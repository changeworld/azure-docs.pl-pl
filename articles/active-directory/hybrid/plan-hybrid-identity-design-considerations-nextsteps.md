# <a name="azure-active-directory-hybrid-identity-design-considerations--next-steps"></a>Azure Active Directory hybrydowego tożsamości projektu dalej zagadnienia dotyczące kroki
Po ukończeniu zdefiniowaniu wymagań i sprawdzeniu wszystkich opcji rozwiązania do zarządzania urządzeniami przenośnymi, możesz wykonać kolejne kroki w celu wdrożenia infrastruktury pomocniczej, który jest odpowiedni dla Ciebie i Twojej organizacji.

## <a name="hybrid-identity-solutions"></a>Rozwiązania tożsamości hybrydowej
-Wykorzystanie określonych scenariuszy rozwiązań, które odpowiadają potrzebom to doskonały sposób na zweryfikowanie i zaplanowanie szczegółów wdrażania infrastruktury zarządzania urządzeniami przenośnymi. Następujące rozwiązania przedstawiają kilka najbardziej typowych scenariuszy zarządzania urządzeniami przenośnymi:

* [Zarządzanie urządzeniami przenośnymi i komputerami w środowisku przedsiębiorstwa](https://technet.microsoft.com/library/dn582037.aspx) ułatwia zarządzanie urządzeniami przenośnymi dzięki rozszerzeniu lokalnej infrastruktury programu System Center 2012 Configuration Manager do chmury w usłudze Microsoft Intune. Ta infrastruktura hybrydowa pomaga specjalistom IT w środku i dużych środowiskach włączać model BYOD i dostępu zdalnego, jednocześnie zmniejszając złożoność administrowania.
* [Zarządzania urządzeniami przenośnymi dla programu Configuration Manager 2007 rozwiązania](https://technet.microsoft.com/library/dn508400.aspx) ułatwia zarządzanie urządzeniami przenośnymi, gdy infrastruktura opiera się na programie System Center Configuration Manager 2007. To rozwiązanie przedstawia sposób konfigurowania pojedynczego serwera z programem System Center 2012 Configuration Manager, dzięki czemu można uruchomić programu Microsoft Intune i wykorzystać jego możliwości zarządzania urządzeniami Przenośnymi.
* [Zarządzania urządzeniami przenośnymi w małych środowiskach rozwiązania](https://technet.microsoft.com/library/dn715906.aspx) jest przeznaczona dla małych firm, które potrzebują obsługi zarządzania urządzeniami przenośnymi. Wyjaśniono, jak rozszerzyć bieżącą infrastrukturę do obsługi zarządzania urządzeniami przenośnymi i modelu BYOD za pomocą programu Microsoft Intune. To rozwiązanie przedstawia najprostszy obsługiwany scenariusz użycia Microsoft Intune w autonomicznej konfiguracji tylko w chmurze bez serwerów lokalnych.

## <a name="hybrid-identity-documentation"></a>Dokumentacja tożsamości hybrydowej
Pojęciach i procedurach planowania, wdrażania i zawartości administracyjnej są przydatne podczas implementowania rozwiązania do zarządzania urządzeniami przenośnymi:

* [Microsoft System Center](https://technet.microsoft.com/library/cc507089.aspx) rozwiązania mogą pomóc Ci zdobycie i zgromadzenie informacji na temat infrastruktury, zasad, procesów i najlepszych rozwiązań, aby działu IT będą mogli stworzyć systemy do zarządzania i zautomatyzować operacje.
* [Microsoft Intune](https://technet.microsoft.com/library/jj676587.aspx) to usługa zarządzania urządzeniami w chmurze, która ułatwia zarządzanie komputerami i urządzeniami przenośnymi oraz zabezpieczanie informacji firmy.
* [Usługa MDM dla Office 365](https://technet.microsoft.com/library/ms.o365.cc.devicepolicy.aspx) pozwala na zarządzanie i zabezpieczanie urządzeń przenośnych, gdy są one połączone z organizacją usługi Office 365. Usługa MDM dla Office 365 służy do konfigurowania zasad i reguły dostępu zabezpieczeń urządzeń i czyszczenie urządzeń przenośnych, jeśli zostaną zgubione lub skradzione.
